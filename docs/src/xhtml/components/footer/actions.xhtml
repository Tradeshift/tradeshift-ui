Footer actions

This is an alpha preview. All features and APIs can be expected to change!

Whenever a selection offers manipulation options, you can mount the relevant actions much like you would with the buttons. The actions method both sets and gets the actions. If you omit the argument, you"ll get the current actions (jQuery style).

In real life, the actions should of course only appear whenever the user has selected something with the mouse or keyboard. We will just have to pretend that he did that.

You can use array methods like push, pop, shift, unshift, splice and so on to manage tabs, just note that the actions length is a readonly property. For consistency reasons, actions should always have both an icon and a label, much unlike the buttons.

Status message

We can configure the actionbar with a status message. When a selection is made, the massage could be something like "23 products selected". When nothing is selected, it could perhaps be something like "Showing 10-20 out of 123 products".
